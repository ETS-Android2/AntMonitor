Class TunNativeInterface

  • java.lang.Object
    • edu.uci.calit2.antmonitor.lib.vpn.TunNativeInterface

  • class TunNativeInterface
extends java.lang.Object
    This class provide a way to poll the TUN interface and read packets from it. - This implementation is required to support polling read from TUN interface on Android versions below 5.0. - Note that Android 5.0+ already supports polling with Java API, however the performance is not good enough due to polling and reading are done in two separate calls to native code. Created by Anh Le

      • pollRead

        public int pollRead(long timeout,
                    java.nio.ByteBuffer buffer,
                    int bufferHeaderSize)
        Poll and read the TUN interface. The poll and read is carried out in native code. If there is data within timeout, then return the number of bytes read, the bytes read are also stored in mBuffer. The caller can access the bytes read by calling getBuffer(). If there is no data within timeout, return 0.
        Parameters:
        timeout - The timeout duration for the poll in ms.
        buffer - The buffer to read the data to
        bufferHeaderSize - The size of the header (already pre-filled) in the buffer
        Returns:
        The number of bytes read.
